CoffeeScript 继承的学习笔记

首先得坦白，我接触 JavaScript 仅一年，并一直依赖 jQuery。

在这 Node.js 流行的年代， JS 不能只被用作为工具 ，而是作为主开发语言，这对我（或许是你）来说其实是坏消息，因为 JS 好另类，要钻研各种黑魔法才能掌握。

所幸的是 Teambition 用 CoffeeScript 来写 JS 代码，节省了钻研黑魔法的时间。在 CoffeeScript vs. ECMAScript 6 幻灯片中证明了 CS 是优秀的，因此被 ES6 借鉴了很多东西过去～

extend = function(child, parent) {
  for (var key in parent) {
    // 拷贝非原型链上的属性
    if (hasProp.call(parent, key)) child[key] = parent[key];
  }
 
  // 原型继承核心，用 ctor 作为桥
  function ctor() {
    this.constructor = child;
  }
  ctor.prototype = parent.prototype;
  // 实例化桥并用本地 constructor 覆盖原型上的
  child.prototype = new ctor();
 
  // 把超类的原型挂到子类的 __super__ 属性上，方便方法扩张，而不是覆盖
  child.__super__ = parent.prototype;
 
  return child;
}

extend 这个函数在 closure 里面执行，然后再返回继承后的 class。

Amarela = (function(superClass) {
  // 闭包内执行
  extend(Amarela, superClass);
 
  function Amarela() {
    // super 关键词会调用 superClass 的同名方法
    Amarela.__super__.constructor.apply(this, arguments);
  }
  
  // class 里面定义的方法都会被挂到 prototype 上面
  // 如果想定义类方法可以用 className.methodName 在 class 关键词外面
  Amarela.prototype.skinColor = 'Yellow';
  // 返回继承后的类
  return Amarela;
 
})(Human);

搞懂以后发现概念与原理没有那么难，但离实际使用（什么时候用，怎么复用）还有一段距离，所以继续加油咯 : )
