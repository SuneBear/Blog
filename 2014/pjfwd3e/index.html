Professional JavaScript For Web Developers 3rd Edition 读书笔记

可能在看这本书之前，在各大社区看到对这本书的各种好评，于是打算在看这本书的时候写写记录。整本书前十五章看得非常认真也学到了很多东西，后十章没有耐心去看了，就草草看了几眼。不过个人认为前十五章很重要，后十章讲了很多api与新的东西，大可不必牢记它们，只需了解即可，正真用的时候凭借自己的印象去查api文档即可。这本书整体不错，不过对于中国人的第一本入门书籍算难的，所以在看这本书之前还需要有javascript的基础知识。

/*

1.基本概念

javaScript 区分大小写，函数名不能为“关键字（keyword）”。

<script>标签引入外部文件后，就不能在标签中插入代码，因为会被忽略。

**/





/*

2.标识符（identifier）

“标识符”是指变量、函数、属性的名字，或者是函数的参数，开头必须为字母、_、$，采用驼峰法写格式。

标识符不能单独出现这些保留字：implements,interface,let,package,private,public,static,yield

标识符不能单独出现这些关键字：break,case,catch,continue,default,delete,do,else,finally,for,function,if,in,instanceof,new,return,switch,this,throw,try,typeof,var,void,while,with

**/







/*

3. 严格模式（strict mode）

"use strict";

如果在语法检测时发现语法问题，则整个代码块失效，并导致一个语法异常。

如果在运行期出现了违反严格模式的代码，则抛出执行异常。

**/







/*

4.变量（variable）

javascript的变量是一种松散的变量，可以保存任何类型的数据。

用var操作符（关键字），加上变量名（标识符）。

var name = "sune";

在函数里面定义的变量，叫做“局部变量（local variable）”，函数退出后就会被销毁，如果省略var操作符会是变量变成全局的，不过不建议这样。

定义多个变量时，在变量名之间用逗号隔开，JS多少表并列关系时都是用逗号隔开。

**/





/*

5.数据类型（data type）

underfined：没有申明、初始化变量，那么变量的值就是underfined，使用typeof函数返回数据类型。

null：表示一个空对象的指针，会返回object，定义null值就是表明未知。

boolean：只有两个值true与false，使用Boolean()可以转换数据类型。

number：0开头为八进制，0x开头为十六进制，n/a为 not applicable 意思是“不适用”，不要测试某个特效浮点的值，浮点最高进度为17位，

        NaN为 not a number 是一个特殊的数值，表示本来要返回数值的操作却没有返回数值，这样会防止抛出错误，使用isNaN()函数检测。

        parseInt()函数可解析一个字符串，并返回一个整数parseInt("1f",16);返回 31 (16+15)。

string：包含在引号中的unicode字符，字符字变量：\n换行  \t制表  \b空格 \r回车 \f进纸  \\斜杠  \'单引号  \"双引号  

object：一组数据和功能的集合，var name = new object(); 其他所有类型的object继承了基本的行为

**/





/*

6.操作符（operator）

unary：只能操作一个值，包括递增，递减，加减操作。

bitwise：按位与，按位或，按位异或，求反，左移，逻辑右移，算术右移（要考虑符号）。

boolean：not(!),and(&&),or(||)，在与、或时有多个操作数。

multiplicative：multiply(*), divide(/), modulus(%)。

additive：add(+)，当两个操作数有一个是字符串时，结果为字符串。subtract(-)，操作数中若有一个是字符串，则转换成数字，如果失败结果为NaN。

relational：less-than (<), greater-than (>), less-than-or-equal-to (<=), and greater-than-or-equal-to (>=)。

equality：equal(==),not equel(!=)，identically equal(===),not identically equal(!==)。

conditional：variable = boolean_expression ? true_value : false_valuea 三元。

assignment；multiply/assign (*=),add/assign (+=)。

comma：var num1=1, num2=2, num3=3;。

**/

for-in：for (property in expression) statement; for (var propName in window) {document.write(propName);}



label：label: statement;

with：with (expression) statement,expression为对象名，在语句的变量赋值中就可以省略掉对象节点;

switch：switch (expression) {case value: statement break; case value: statement break; default: statement};

12.创建自定义类型（创建对象）

工厂模式：实例化对象，用工厂方法代替new操作 var person = new Object(); person.name = “Nicholas”;



构造函数模式：第一个字母大写，没有rutuen语句，直接将属性或者方法赋给this对象。

其实构造函数就是普通的函数，只是在函数前加一个new操作符就可以创建一个对象了，如果不用new那么在使用时属性与方法会被添加给window。

在多个实例用公用方法时会造成重复。



原型模式：其实原型模式也是通过调用构造函数而去创建那个对象的原型对象，在创建任何函数的时候，都会有prototype属性，该属性还会自动获取一个constructor属性，isPrototypeOf()检测实例是否有指针，有则返回true。在原型实例获取属性是，先去访问实例时候含有属性，然后再去访问原型。hasOwnProperty()方法判断属性存在实例还是原型。原型具有动态性，因为在实例中没有找到的话就会访问原型。但在重写整个原型后就会出错，调用构造函数时会为实例添加一个指向最初原型的__proto__指针，



组合模式：结构构造函数与原型的优点，方法与共享属性用原型。



动态原型模式：使用if判断语句添加原型方法，如果存在则不添加



寄生构造函数模式：一般不建议使用



稳妥构造函数模式：安全性高

13.继承（inheritance）

js中只支持“实现继承”，不能实现“接口继承”。每一个对象都会包含constructor属性，模式是指向生成它的“类”，在实例中prototype变成__proto__。

也就说prototype是一种属性，而__proto__只是一种指针。



原型链继承：原型之间的指向联系，实例-->原型对象-->object超类  SonType.prototype = new FatherType(); 这样子类的原型就变成了父类的原型。在实际使用中很少会去单独用原型链继承，因为原型链不能向超类型的构造函数传递参数，并且原型中含有引用类型的值时，更改会影响实例



借用构造函数继承：function SubType(){ SuperType.call(this);} call里面可以填写传递的参数。这样方式因为方法都在构造函数中定义，所以就没有函数复用可言。



组合继承：有时候也叫为经典继承，把以上另种方式是结合。原型链用来继承原型的方法与属性，借用构造函数来实现实例属性的继承。



原型式继承：function object(o){  function F(){}  F.prototype = o;  return new F(); },表示不懂。



寄生式继承：创建一个仅用于封装继承过程的函数，该函数在内部以某种方式来增强对象，最后再像真的是他做了所有工作一样返回对象。



寄生组合式继承：......好像和组合继承类似的

14.函数表达式（function anonymous）

匿名函数：在function方法中面没有标识符，定义要在执行之前， 而普通的函数定义则不需要这样。



递归：函数调用自身，常使用arguments.callee()方法



闭包：闭包就是能够读取其他函数内部变量的函数。由于在Javascript语言中，只有函数内部的子函数才能读取局部变量，因此可以把闭包简单理解成“定义在一个函数内部的函数”。所以，在本质上，闭包就是将函数内部和函数外部连接起来的一座桥梁。  //先跳过，回头再来理解......



块级作用域：任何一对花括号（｛和｝）中的语句集都属于一个块，在这之中定义的所有变量在代码块外都是不可见的，我们称之为块级作用域。在块级外面可以重新声明变量来模拟块级作用域。 



私有变量：在js中没有私有成员的概念，但函数中定义的变量可以认识为是私有变量。在把私有变量变成共有变量的方法叫做特权方法。

var EventUtil = {

addHandler: function(element, type, handler){

if (element.addEventListener){

element.addEventListener(type, handler, false);

} else if (element.attachEvent){

element.attachEvent(“on” + type, handler);

} else {

element[“on” + type] = handler;

}

},

removeHandler: function(element, type, handler){

if (element.removeEventListener){

element.removeEventListener(type, handler, false);

} else if (element.detachEvent){

element.detachEvent(“on” + type, handler);

} else {

element[“on” + type] = null;

}

}

};

对做笔记的思考

把这本书的笔记做完后，我对笔记的作用产生了疑惑，感觉它的作用不是很大。记得高中每次看书都要做笔记，整个高中下来记录了很多笔记，结果高中毕业后整个笔记就被闲置了，我想以后也不会再去看了。所以决定看书要少做笔记，也不要刻意要求自己去做笔记，不然就像被限制住了，得不到看书的自由。感觉很多东西互联网上都有，为什么还要记一次，如果记下来后还是不记得，那么我记的笔记与互联网上的知识又有何异？坑爹的一段话我又扯远了，我只是来上传笔记的…
